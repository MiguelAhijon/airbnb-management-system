# 🏠 Sistema de Gestión de Reservas tipo Airbnb

Proyecto académico desarrollado como simulación de una plataforma tipo Airbnb, orientada a la gestión de usuarios, alojamientos, reservas y procesos asociados.

## 🚀 Descripción

El proyecto consiste en el análisis, diseño y desarrollo de un sistema de información inspirado en Airbnb, aplicando metodología ágil Scrum y utilizando GitHub para la organización del trabajo.

Se trabajó sobre el ciclo completo del proyecto: requisitos, modelado, diseño, planificación, gestión de tareas y control de versiones.

## 🎯 Objetivos

- Diseñar un sistema de gestión de reservas.
- Modelar usuarios, alojamientos y reservas.
- Aplicar una metodología ágil basada en Scrum.
- Gestionar el proyecto mediante issues, sprints y tareas en GitHub.
- Simular un entorno real de desarrollo colaborativo.

## 🧠 Funcionalidades principales

- Gestión de usuarios.
- Gestión de alojamientos.
- Gestión de reservas.
- Definición de requisitos funcionales.
- Modelado de base de datos.
- Organización del trabajo mediante Scrum.
- Seguimiento de tareas con GitHub Issues.

## 🛠️ Tecnologías y herramientas

- Git
- GitHub
- GitHub Issues
- Scrum
- SQL
- Modelado de datos
- Diseño de sistemas de información

## 🗂️ Metodología Scrum

El proyecto se organizó siguiendo una metodología ágil Scrum:

- Definición del Product Backlog.
- Planificación de sprints.
- Creación y seguimiento de issues.
- Reparto de tareas.
- Revisión del avance del proyecto.
- Control de versiones mediante GitHub.

## 🧩 Modelo de datos

El sistema contempla entidades como:

- Usuarios
- Alojamientos
- Reservas
- Pagos
- Valoraciones

Estas entidades permiten representar el funcionamiento básico de una plataforma de alquiler de alojamientos.

## 📌 Aprendizajes

Este proyecto permitió reforzar conocimientos sobre análisis y diseño de sistemas de información, bases de datos, metodologías ágiles y trabajo colaborativo con GitHub.
